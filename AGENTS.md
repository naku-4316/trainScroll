# AGENTS.md

## Language
- 会話、要約、作業報告、PRタイトル、PR説明、コメントは必ず日本語で書いてください。
- ユーザー向けの文面は英語にしないでください。
- コード内コメントも、特別な理由がなければ日本語で書いてください。

## Project workflow
- 必ず最新の `main` ブランチを土台にして作業してください。
- 1タスク = 1機能 = 1PR を基本にしてください。
- すでにマージ済みの古い作業ブランチを使い回さないでください。
- 大きな変更をまとめて入れず、小さな差分で進めてください。
- 指示がない限り、リポジトリ設定や GitHub Pages 設定は変更しないでください。

## Files to read before work
作業前に必ず次のファイルを読んでください。

- `PROJECT_NOTES.md`
- `TASK_HISTORY.md`

これらを読み、現在仕様・制約・過去の作業履歴を理解してから変更してください。

## Files to maintain
作業後は必要に応じて次のファイルを更新してください。

### `TASK_HISTORY.md`
毎回の作業後に、以下を日本語で追記してください。

- 日付
- 今回やったこと
- 変更したファイル
- 残課題
- 次の推奨タスク

### `PROJECT_NOTES.md`
仕様や方針が変わった場合のみ更新してください。  
一時的な作業ログは書かず、現在有効な仕様だけを簡潔に保ってください。

## Scope control
- 今回の指示に直接関係ないファイルは変更しないでください。
- 指示されていない README の更新はしないでください。
- 不要なリファクタリングはしないでください。
- 既存の公開ページが壊れる可能性がある変更は慎重に扱ってください。

## Implementation policy
- まず動く最小実装を優先してください。
- 見た目の全面改修より、指示された機能達成を優先してください。
- 既存挙動を壊さないことを重視してください。
- 変更理由が分かるように、コードは読みやすく保ってください。

## Validation
作業後は、可能な範囲で最低限の確認を行ってください。

- 構文エラーがないこと
- 明らかな実行エラーがないこと
- 変更内容が今回の依頼に一致していること

確認した内容は作業報告に日本語で書いてください。

## When unclear
- 仕様が曖昧な場合は、勝手に大きく拡張せず、最小限で安全な実装にしてください。
- 大きく構造変更する前に、まず小さい差分案を優先してください。

## Pull requests
- PRタイトルと説明は日本語で書いてください。
- PR説明には以下を簡潔に含めてください。
  - 目的
  - 変更内容
  - 確認内容
  - 残課題

## Important reminder
- このプロジェクトでは、過去タスクを同じ作業ブランチのまま継続しないでください。
- 新しい作業は常に最新の `main` を土台にしてください。
- 作業前に `PROJECT_NOTES.md` と `TASK_HISTORY.md` を読み、作業後に `TASK_HISTORY.md` を更新してください。
